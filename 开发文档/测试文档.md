# 测试文档

## 测试目标

当前测试覆盖 Feed 主链路、分页加载、基础播放状态、搜索中间页、搜索结果页、推荐词入口，以及“视频 Feed -> 搜索 -> 搜索结果 -> 回到 Feed 定位播放”的视频全链路，确保 MVP 的核心交互可以稳定运行。

## 2. Feed 分页加载测试

测试文件：

## 3. 基础播放状态测试

测试文件：

[feed_player_test.dart](D:/Flutter/VideoPlayer/test/feed_player_test.dart:1)

测试内容：

- App 启动后首个内容是视频卡
- `playerControllerProvider` 当前 `videoId` 为 `video_001`
- 视频页存在进度条 `Slider`
- 视频页存在播放按钮
- 上滑切换到图片卡
- 图片卡标题 `雨后城市天台的晚霞` 可以展示
- 切换到图片卡后播放器停止
- `playerControllerProvider` 当前 `videoId` 变为 `null`

通过标准：

```text
视频卡进入页面时会触发播放器状态。
离开视频卡进入图片卡时会停止播放器。
```

## 7. 视频全链路测试

测试文件：

[video_full_flow_test.dart](D:/Flutter/VideoPlayer/test/video_full_flow_test.dart:1)

测试内容：

- 使用 fake video platform 模拟视频插件初始化、播放、暂停和播放位置
- App 启动后首个视频自动进入播放状态
- `playerControllerProvider.videoId` 为 `video_001`
- 首屏展示 `5 分钟学会篮球变向运球`
- 点击搜索入口时暂停当前播放器
- 在搜索中间页输入 `手冲` 并提交
- 搜索结果页展示扩充 mock 数据中的 `手冲咖啡入门：稳定萃取三件事`
- 点击搜索结果项
- 自动加载并定位 Feed 中的 `video_005`
- 回到 Feed 后播放器状态切换到 `video_005`
- 目标视频完成初始化并进入播放状态
- 调用播放/暂停切换后播放器进入暂停状态

通过标准：

```text
用户可以从 Feed 进入搜索，选择不在首屏分页中的视频结果，
系统会加载更多 Feed 数据、定位目标视频，并启动对应视频播放。
```
