# 2 图文页多图片支持缺失

## 当前问题

图文 Feed 原本按单张图片设计，`ImageFeedItem` 只保存一个 `imageUrl`，图文卡片也只渲染一张图片。

这种设计无法表达常见的多图内容场景：

- 单个图文 Feed 无法承载多张图片。
- 用户不能在同一条图文内容内横向浏览图片。
- mock 数据中的图文图片数量固定，无法覆盖真实业务中 3 张、4 张、5 张等变长图集。
- 测试断言依赖固定图片数量，不利于后续扩展。

## 优化目标

将图文 Feed 从单图模型升级为多图模型，并支持单个 Feed 页面内横向浏览多张图片。

优化后应达到：

- `ImageFeedItem` 支持多张图片地址。
- 图文卡片内部支持横向分页浏览图片。
- 页面能显示当前图片页码和底部分段指示器。
- mock 数据支持不同图片数量，贴近真实图文内容。
- 测试不依赖固定图片数量，而是按当前数据动态断言。

## 优化范围

本次优化涉及图文数据模型、mock 数据、图文卡片 UI 和相关测试。

核心代码范围：

- [image_feed_item.dart](d:/Flutter/VideoPlayer/lib/data/models/image_feed_item.dart)：`imageUrl` 改为 `imageUrls`，并保留 `imageUrl` getter 兼容旧引用。
- [mock_images.dart](d:/Flutter/VideoPlayer/lib/mock/mock_images.dart)：图文 mock 数据改为多张图片，并支持 3 张、4 张、5 张等不同数量。
- [image_feed_card.dart](d:/Flutter/VideoPlayer/lib/features/feed/widgets/image_feed_card.dart)：图文区域改为内部横向 `PageView`。
- 相关 widget 测试：收紧外层竖向 `PageView` 查找，并增加图文横滑和变长 mock 数据验证。

## 优化方案

将图文数据结构从单图片字段升级为图片列表：

```text
imageUrl
  -> imageUrls
```

为降低旧代码迁移成本，保留 `imageUrl` getter，让仍然读取单图字段的旧引用可以拿到第一张图片。

图文卡片内部增加横向 `PageView`：

```text
Feed 外层竖向 PageView
  -> ImageFeedCard
  -> 内部横向 PageView
  -> 多张图片
```

UI 上补充：

- `图文 1/3` 这类当前页码提示。
- 底部分段指示器，用于表达多图进度。
- 渐变遮罩忽略手势，避免遮挡内部横向滑动。

mock 数据不再统一固定图片数量，而是提供不同长度的 `imageUrls`，用于覆盖更接近真实业务的数据形态。

## 优化验证

从数据结构验证：

- `ImageFeedItem` 可以保存多张图片。
- 旧的 `imageUrl` getter 仍然可返回首张图片，兼容旧引用。
- mock 数据中存在不同图片数量的图文内容。

从交互行为验证：

- 单个图文 Feed 内可以横向滑动切换图片。
- 外层 Feed 仍然可以纵向切换内容。
- 渐变遮罩不会拦截图片横滑手势。

从测试覆盖验证：

- 图文横滑测试验证内部 `PageView` 可切换图片。
- 页码断言按当前 mock item 的 `imageUrls.length` 动态计算。
- 测试确认 mock 数据中包含 `3/4/5` 等不同图片数量。
