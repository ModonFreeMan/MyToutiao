# 14.1 搜索离线索引与 Embedding 入库

## 当前目标

在 `14搜索链路完整设计方案` 的基础上，先完成搜索链路的离线部分。

本阶段只处理视频入库后的离线索引能力：

```text
读取视频标题、简介、标签和推荐词；
生成视频内容摘要；
生成 5 到 10 个搜索关键词；
调用 embedding 模型生成摘要向量；
把可读搜索信息写入业务库；
把摘要向量写入 Milvus；
Milvus 向量索引使用 FLAT。
```

本阶段不做在线搜索接口、查询词 embedding、向量召回、关键词召回和融合排序，这些放到后续在线链路实现。

## 实现边界

当前视频摘要暂时不调用大模型生成，而是使用固定描述：

```text
这个视频主要介绍 Flutter 视频播放器的横屏播放和清晰度切换。
```

关键词由模型生成，输入为本地视频信息：

```text
title
description
tags
recommendationWords
transcript
```

模型只生成推荐词，不生成摘要。摘要仍使用本地视频信息文本，避免所有视频写入相同摘要后导致 Milvus 向量缺少区分度。

模型返回的推荐词会去重后最多保留 10 个；如果模型返回不足，会用本地 title、tags、recommendationWords 补齐。

职责划分：

```text
OfflineVideoInput
  离线入库输入，包含 videoId、title、description、tags、recommendationWords、transcript。

SearchVideoDocument
  业务库搜索文档，包含 videoId、title、summary、keywords。

FixedVideoSummaryGenerator
  临时摘要和关键词生成器，摘要写死，关键词从 mock 视频信息中整理。

OpenAICompatibleSearchEmbeddingService
  OpenAI compatible embedding 客户端。

ChatCompletionKeywordVideoSummaryGenerator
  OpenAI compatible chat completions 客户端，只生成推荐词，不生成摘要。

JsonSearchBusinessStore
  本地 JSON 业务库，用于保存可读搜索信息。

MilvusSearchVectorStore
  Milvus 向量写入器，负责创建 collection 和写入 summaryEmbedding。

SearchOfflineConfig
  本地配置读取，注入 API 端点、API key、模型、Milvus 地址和业务库路径。

VideoSearchOfflineIndexer
  离线索引编排器，串联摘要、embedding、业务库和 Milvus。
```

离线脚本：

```text
tool/build_search_offline_index.dart
```

脚本默认读取 mock 视频数据，并将视频写入本地业务库和 Milvus。

## Embedding 配置

embedding 使用：

```text
模型：text-embedding-3-small
中转 API Base URL：从 config.search_offline_config.json 的 apiBaseUrl 读取
中转 Embedding 端点：从 config.search_offline_config.json 的 embeddingEndpoint 读取
```

推荐词生成使用：

```text
模型：从 config.search_offline_config.json 的 chatModel 读取
接口：从 config.search_offline_config.json 的 chatCompletionEndpoint 读取
默认模型：gpt-4o-mini-2024-07-18
接口类型：/v1/chat/completions
```

配置文件路径：

```text
config/search_offline_config.json
```

示例配置文件：

```text
config/search_offline_config.example.json
```

真实配置文件包含 API key，因此已经加入 `.gitignore`：

```text
/config/search_offline_config.json
```

示例配置：

```json
{
  "apiBaseUrl": "replace-with-api-base-url",
  "chatCompletionEndpoint": "replace-with-chat-completions-endpoint",
  "chatModel": "gpt-4o-mini-2024-07-18",
  "embeddingEndpoint": "replace-with-embedding-endpoint",
  "apiKey": "replace-with-your-api-key",
  "embeddingModel": "text-embedding-3-small",
  "milvusHost": "replace-with-milvus-host",
  "milvusPort": 19530,
  "milvusCollectionName": "video_summary_search_flat",
  "businessStorePath": "build/search/video_documents.json"
}
```

注意：真实 API key 只写入本地 `config/search_offline_config.json`，不提交到 git。

## Milvus 设计

Milvus 地址从本地配置文件读取：

```text
milvusHost
milvusPort
```

本次的 collection：

```text
video_summary_search_flat
```

collection 字段：

```text
videoId
  VarChar
  primary key
  max_length = 128

summaryEmbedding
  FloatVector
  dim = embedding 返回维度
```

向量索引：

```text
index_type = FLAT
metricType = COSINE
fieldName = summaryEmbedding
indexName = summaryEmbedding_flat
```

当前 collection 创建时直接携带 `indexParams`，保证初始索引就是 FLAT，避免 Milvus 自动创建默认索引后无法切换的问题。

## 离线脚本

运行命令：

```text
dart run tool/build_search_offline_index.dart --limit 10
```

可选参数：

```text
--config
  指定配置文件路径，默认 config/search_offline_config.json。

--business-store
  覆盖业务库 JSON 输出路径。

--milvus-host
  覆盖 Milvus host。

--milvus-port
  覆盖 Milvus port。

--milvus-collection
  覆盖 Milvus collection。

--limit
  指定处理前 N 条 mock 视频。
```

当前脚本默认处理 `mockFeedItems` 中的视频项，跳过图文项。

业务库输出路径：

```text
build/search/video_documents.json
```

该文件是运行产物，位于 `build/` 下，不提交到 git。

## 后续工作

下一阶段可以继续实现在线搜索链路：

```text
1. 用户搜索词调用 text-embedding-3-small 生成 query embedding。
2. 查询 Milvus 的 video_summary_search_flat collection。
3. 从业务库读取 title、summary、keywords。
4. 实现关键词检索：
   title 命中 +5
   keywords 命中 +4
   summary 命中 +1
5. 合并向量结果和关键词结果。
6. 使用 finalScore = vectorScore * 0.7 + keywordScore * 0.3 排序。
7. 对同一个 videoId 去重。
8. 返回最终视频列表。
```

后续替换点：

```text
FixedVideoSummaryGenerator
  替换为真实 AI 摘要和关键词生成器。

JsonSearchBusinessStore
  替换为真实业务库读写。

在线 SearchDataSource
  替换当前 MockSearchDataSource 或新增真实搜索数据源。
```
