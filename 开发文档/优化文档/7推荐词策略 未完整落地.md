# 7 推荐词策略未完整落地

## 当前问题

推荐词业务逻辑原本只实现了部分匹配策略：

```text
recommendationWords -> tags -> default
```

但设计文档中还包含基于 `title` 和 `description` 的文本匹配逻辑。当前实现缺少这两层判断，会导致部分内容即使标题或描述里包含可推荐关键词，也无法命中对应推荐词，只能退回到默认推荐。

这会让推荐词结果不够贴合当前内容，也会让代码实现和设计文档出现偏差。

## 优化目标

补齐推荐词匹配策略，让推荐词生成顺序完整覆盖显式推荐词、标签、标题、描述和默认兜底。

优化后策略为：

```text
recommendationWords -> tags -> title -> description -> default
```

优化后应达到：

- 优先使用内容自身携带的 `recommendationWords`。
- 没有显式推荐词时，优先按 `tags` 匹配。
- `tags` 无结果时，继续按 `title` 匹配。
- `title` 无结果时，继续按 `description` 匹配。
- 所有策略都无结果时，才使用默认推荐词。

## 优化范围

本次优化涉及推荐词服务、推荐词数据源接口、Mock 数据源、Repository 和单元测试。

核心代码范围：

- [recommendation_service.dart](d:/Flutter/VideoPlayer/lib/features/recommendation/services/recommendation_service.dart:11)：补齐 `title`、`description` 匹配顺序。
- [recommendation_data_source.dart](d:/Flutter/VideoPlayer/lib/data/datasources/recommendation_data_source.dart:4)：新增 `fetchWordsByText`。
- [mock_recommendation_datasource.dart](d:/Flutter/VideoPlayer/lib/data/datasources/mock_recommendation_datasource.dart:20)：用现有推荐词库按文本关键词匹配。
- [recommendation_repository.dart](d:/Flutter/VideoPlayer/lib/data/repositories/recommendation_repository.dart:29)：透传文本匹配方法。
- [recommendation_service_test.dart](d:/Flutter/VideoPlayer/test/recommendation_service_test.dart:12)：新增优先级相关单元测试。

## 优化方案

在 `RecommendationService` 中统一编排推荐词策略，按固定优先级逐层尝试：

```text
1. recommendationWords
2. tags
3. title
4. description
5. default
```

当上一级策略返回有效结果时，直接使用该结果，不继续向下匹配。

为支持标题和描述匹配，在 `RecommendationDataSource` 中增加文本匹配能力：

```dart
Future<List<String>> fetchWordsByText(String text);
```

`MockRecommendationDataSource` 使用现有推荐词库按文本关键词进行匹配，`RecommendationRepository` 负责把该能力向上透传给 Service。

整体调用关系保持为：

```text
RecommendationService
  -> RecommendationRepository
  -> RecommendationDataSource
  -> MockRecommendationDataSource
```

## 优化验证

从策略顺序验证：

- 存在 `recommendationWords` 时，直接使用显式推荐词。
- `recommendationWords` 为空时，使用 `tags` 匹配结果。
- `tags` 无匹配时，使用 `title` 匹配结果。
- `title` 无匹配时，使用 `description` 匹配结果。
- 所有内容字段都无匹配时，使用默认推荐词。

从测试覆盖验证：

- `recommendation_service_test.dart` 已覆盖推荐词优先级链路。
- 测试应能证明后级策略不会覆盖前级策略。
- 测试应能证明默认推荐只在所有策略无结果时生效。

从文档一致性验证：

- 当前实现已补齐设计文档中的 `title`、`description` 匹配逻辑。
- 推荐词业务链路从 Service 到 DataSource 的职责划分保持清晰。
- 后续接入真实推荐接口时，可以复用 `fetchWordsByText` 抽象能力。
