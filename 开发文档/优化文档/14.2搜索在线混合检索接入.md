# 14.2 搜索在线混合检索接入

## 当前目标

在 `14搜索链路完整设计方案` 和 `14.1搜索离线索引与Embedding入库` 的基础上，完成在线搜索链路的最小可用版本。

本阶段处理：

```text
读取用户 query；
调用 embedding 模型生成 query embedding；
使用 query embedding 查询 Milvus，得到 vectorScore；
读取业务搜索文档；
按 title、keywords、summary 做关键词检索，得到 keywordScore；
按固定权重合并 vectorScore 和 keywordScore；
把最终排序结果接入项目搜索结果页。
```

本阶段不做：

```text
低分阈值过滤；
真实后端业务库接入；
复杂分词、同义词、拼音、纠错；
搜索解释信息展示；
分页和加载更多。
```

## 在线流程

输入：

```text
用户搜索词
```

输出：

```text
List<VideoFeedItem>
```

处理流程：

```text
query.trim()
-> OpenAICompatibleSearchEmbeddingService.embed(query)
-> MilvusSearchVectorStore.searchSummaryEmbedding(queryEmbedding)
-> JsonSearchBusinessStore.findAll()
-> 对业务文档计算 keywordScore
-> 合并 vectorScore 和 keywordScore
-> 按 finalScore 倒序排序
-> 按 videoId 映射成 VideoFeedItem
```

如果 query 为空字符串，直接返回空结果，不调用 embedding 模型，也不访问 Milvus。

```text

  使用 Milvus REST 接口 /v2/vectordb/entities/search 查询 summaryEmbedding。

SearchBusinessStore.findAll
  读取全部业务搜索文档，用于本地关键词检索。

VideoDenseSearchService
  在线稠密检索编排器，串联 query embedding 和 Milvus vector search。

DenseSearchDataSource
  项目搜索数据源，负责执行向量检索、关键词检索、融合排序，并返回 VideoFeedItem。

tool/search_dense_query.dart
  本地命令行联调入口，方便直接输入 query 验证 Milvus 稠密检索结果。
```

## 稠密检索

Milvus 查询接口：

```text
/v2/vectordb/entities/search
```

请求核心字段：

```json
{
  "collectionName": "video_summary_search_flat",
  "data": ["queryEmbedding"],
  "annsField": "summaryEmbedding",
  "limit": 10,
  "outputFields": ["videoId"]
}
```

返回解析：

```text
videoId
  优先读取 videoId；
  兼容读取 id；
  兼容读取 entity.videoId。

score
  优先读取 score；
  兼容读取 distance；
  缺失时默认为 0。
```

这里的 `score` 作为后续融合排序中的 `vectorScore`。

## 关键词检索

关键词检索使用 14 总设计中的最简单打分规则：

```text
title 命中：+5
keywords 命中：+4
summary 命中：+1
```

实现位置：

```text
lib/data/datasources/dense_search_datasource.dart
```

当前规则：

```text
query 会 trim 后参与检索；
title 使用 contains 匹配；
keywords 中任意一个词 contains 命中则加一次 +4；
summary 使用 contains 匹配；
大小写统一转 lowercase 后比较；
keywordScore 为 0 的文档不进入关键词召回集合。
```

## 融合排序

融合前先把本次关键词分数归一化：

```text
normalizedKeywordScore = keywordScore / maxKeywordScore
```

如果本次没有任何关键词命中：

```text
normalizedKeywordScore = 0
```

最终分数：

```text
finalScore = vectorScore * 0.7 + normalizedKeywordScore * 0.3
```

合并规则：

```text
向量召回结果和关键词召回结果按 videoId 合并；
同一个 videoId 只保留一条；
只要 vectorScore 或 keywordScore 有一个大于 0，就参与最终排序；
最终按 finalScore 从高到低排序。
```

## 项目接入

项目搜索入口：

```text
lib/data/repositories/search_repository.dart
```

当前 provider：

```text
searchDataSourceProvider -> DenseSearchDataSource.localConfig()
```

页面搜索结果链路保持不变：

```text
SearchResultPage
-> searchResultsProvider(keyword)
-> SearchRepository.searchVideos(keyword)
-> DenseSearchDataSource.searchVideos(keyword)
-> List<VideoFeedItem>
```

这意味着搜索结果页不需要改 UI，也不需要知道底层是向量检索、关键词检索还是融合排序。

## 配置与回退

`DenseSearchDataSource.localConfig()` 默认读取：

```text
config/search_offline_config.json
```

使用配置项：

```text
apiKey
embeddingEndpoint
embeddingModel
milvusHost
milvusPort
milvusCollectionName
businessStorePath
```

当前项目接入带有回退：

```text
如果本地配置缺失；
如果 embedding 请求失败；
如果 Milvus 请求失败；
如果业务搜索文档读取失败；
则回退到 MockSearchDataSource。
```

这样可以避免测试环境、未配置环境或 Milvus 暂不可用时搜索页面直接不可用。

## 离线摘要修正

联调在线稠密检索时，第一次查询发现 10 条结果分数高度接近：

```text
video_006       0.37584394
video_008       0.37583014
video_005       0.37583014
video_002       0.37583014
video_001       0.37583014
video_010       0.3758211
video_009       0.3758211
video_007       0.3758211
video_004       0.3758211
video_003       0.3758211
```

原因：

```text
14.1 阶段为了先跑通入库，离线脚本使用 FixedVideoSummaryGenerator；
所有视频写入 Milvus 的 summary 都是同一段固定文本；
因此不同视频的 summaryEmbedding 几乎没有区分度。
```

本次修正：

```text
新增 LocalVideoSummaryGenerator；
使用 video.title、description、tags、recommendationWords、transcript 拼成本地 summary；
tool/build_search_offline_index.dart 改用 LocalVideoSummaryGenerator；
保留 FixedVideoSummaryGenerator 作为兼容测试和后续对照使用。
```

本地 summary 格式：

```text
标题：{title}
简介：{description}
标签：{tags}
推荐词：{recommendationWords}
字幕：{transcript}
```

该修正不引入大模型，只让不同视频的入库文本先具备最基本的区分度。
