# 13 Feed 主媒体缩放固定裁剪

## 当前问题

进入竖屏 Feed 页面后，主媒体画面被固定铺满展示区域，横版视频和非竖版图片无法完整显示。

典型表现是：

```text
视频能正常播放；
但画面被放大裁剪；
图片能正常加载；
但用户看不到完整媒体内容；
横版视频和非竖版图片在竖屏页面中尤其明显。
```

当前竖屏播放器视图使用 `FittedBox + BoxFit.cover` 渲染真实视频：

```dart
return FittedBox(
  fit: BoxFit.cover,
  child: SizedBox(
    width: size.width,
    height: size.height,
    child: VideoPlayer(controller),
  ),
);
```

当前图文卡片主图使用 `Image.network + BoxFit.cover` 渲染图片：

```dart
return Image.network(
  imageUrl,
  fit: BoxFit.cover,
);
```

`BoxFit.cover` 的语义是保持媒体宽高比，同时强制铺满父容器。如果媒体比例和父容器比例不同，超出的部分会被裁剪。

Feed 视频卡片和图文卡片的主媒体区域都是接近全屏的竖版容器：

```text
VideoFeedCard
  -> Column
  -> Expanded
  -> Stack(fit: StackFit.expand)
  -> VideoPlayerView

ImageFeedCard
  -> Column
  -> Expanded
  -> Stack(fit: StackFit.expand)
  -> PageView
  -> _FeedImageView
```

而当前 mock 视频源统一按横版尺寸配置：

```text
640 x 360
1280 x 720
1920 x 1080
```

因此实际组合是：

```text
横版视频 / 非竖版图片
  -> 放进竖版全屏播放区域
  -> 使用 BoxFit.cover 铺满
  -> 画面被明显裁剪
```

横屏播放页没有同样的问题。横屏页的视频渲染使用 `BoxFit.contain`，会完整显示视频，只是在比例不匹配时留黑边。

## 优化目标

本次优化目标是让竖屏 Feed 页面完整展示主媒体内容，避免真实视频播放画面和图片画面只能看到被裁剪后的局部区域。

需要满足：

```text
竖屏真实视频播放时，默认能看到完整画面；
竖屏图文主图展示时，默认能看到完整画面；
视频宽高比保持正确，不拉伸变形；
图片宽高比保持正确，不拉伸变形；
未初始化封面继续保持现有铺满展示；
横屏播放页现有完整显示逻辑不受影响；
播放、暂停、进度、清晰度切换、横屏入口等控制逻辑不受影响；
不引入复杂缩放状态机或额外业务状态。
```

本次不追求短视频平台式沉浸裁剪效果，而是保证 MVP 阶段 Feed 主媒体内容可见性。

## 优化范围

本次只修改竖屏 Feed 主媒体画面的缩放策略：

```text
lib/features/player/widgets/video_player_view.dart
lib/features/feed/widgets/image_feed_card.dart
```

重点调整真实视频和图文主图渲染分支：

```text
FittedBox(fit: BoxFit.cover)
Image.network(fit: BoxFit.cover)
```

不修改：

```text
lib/features/player/pages/landscape_player_page.dart
lib/features/player/controllers/player_controller.dart
lib/features/player/states/player_state.dart
lib/features/feed/coordinators/feed_playback_coordinator.dart
lib/features/feed/widgets/video_feed_card.dart
lib/features/feed/widgets/feed_content_info.dart
```

原因是这个问题只发生在 UI 缩放策略层，不属于播放器生命周期、播放状态机、Feed 编排或横屏切换问题。

非主媒体图片保持现状：

```text
真实视频：
  改为 BoxFit.contain，保证完整显示。

图文主图：
  改为 BoxFit.contain，保证完整显示。

未初始化封面：
  继续保持 BoxFit.cover，让 Feed 首屏视觉保持饱满。

作者头像：
  继续保持 BoxFit.cover，维持圆形头像裁剪效果。
```

为避免过度设计，本次只调整 Feed 主媒体，不引入用户可切换的缩放模式。

## 优化方案

### 竖屏主媒体改为完整显示

将 `VideoPlayerView` 中真实视频分支从 `BoxFit.cover` 改为 `BoxFit.contain`：

```dart
return FittedBox(
  fit: BoxFit.contain,
  child: SizedBox(
    width: size.width,
    height: size.height,
    child: VideoPlayer(controller),
  ),
);
```

将 `ImageFeedCard` 中图文主图从 `BoxFit.cover` 改为 `BoxFit.contain`：

```dart
return Image.network(
  imageUrl,
  fit: BoxFit.contain,
);
```

调整后的行为：

```text
横版视频放进竖屏 Feed：
  保持原始比例；
  完整显示视频；
  上下或左右可能出现黑边。

竖版视频放进竖屏 Feed：
  仍可接近铺满；
  不会被额外裁剪。

非竖版图片放进竖屏 Feed：
  保持原始比例；
  完整显示图片；
  上下或左右可能出现黑边。

竖版图片放进竖屏 Feed：
  仍可接近铺满；
  不会被额外裁剪。
```

本次改动只影响竖屏 Feed 主媒体的缩放方式，不改变播放器状态机、播放编排和图文分页逻辑。

### 不纳入本次范围

本次不加入缩放按钮或缩放模式状态，例如：

```text
cover / contain 用户切换；
双击切换填充模式；
按视频比例自动选择不同策略；
在 PlayerState 中保存缩放偏好；
按清晰度或视频源尺寸动态修正布局。
```

当前问题的根因很明确：竖屏 Feed 主媒体使用了固定 `BoxFit.cover`。本次直接改为 `BoxFit.contain`，避免把一个 UI 缩放问题扩大成播放器状态或偏好设置问题。

## 通过标准

本问题优化完成后，需要满足：

```text
竖屏 Feed 中真实视频初始化完成后，可以看到完整播放画面。
竖屏 Feed 中图文主图加载完成后，可以看到完整图片画面。
视频和图片画面不变形。
允许出现黑边。
未初始化封面仍保持铺满裁剪展示。
作者头像仍保持圆形裁剪展示。
横屏页仍然完整显示视频。
播放、暂停、进度条、清晰度切换和横屏入口保持原有行为。
图文多图片横向分页保持原有行为。
不新增播放器状态字段。
不新增用户偏好存储。
```
