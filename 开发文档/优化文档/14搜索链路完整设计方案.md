# 推荐词搜索最简设计方案

## 目标

用最简单的方式完成视频搜索：

- 离线阶段：根据视频内容生成关键词、内容摘要、摘要 embedding，并写入业务库和 Milvus。
- 在线阶段：根据用户搜索词同时做向量检索和关键词检索。
- 排序阶段：把两路结果按简单权重合并，返回最终视频列表。


## 边界

只做：

- 生成视频关键词。
- 生成视频内容摘要。
- 生成视频内容摘要 embedding。
- 摘要 embedding 写入 Milvus。
- 用户搜索词生成 query embedding。
- Milvus 返回稠密检索结果。
- 关键词检索匹配 `title`、`keywords`、`summary`。
- 两路结果按固定权重合并。


## 数据结构

业务库保存视频可读信息：

```json
{
  "videoId": "video_001",
  "title": "Flutter 横屏播放器切换",
  "summary": "这个视频主要介绍 Flutter 视频播放器的横屏播放和清晰度切换。",
  "keywords": ["Flutter 视频播放器", "横屏播放", "清晰度切换"]
}
```

Milvus 保存向量信息：

```json
{
  "videoId": "video_001",
  "summaryEmbedding": [0.12, 0.33, 0.56]
}
```

## 离线服务

离线服务在视频入库或内容更新后执行。

输入：

```text
视频标题
视频简介
视频标签
视频字幕或内容文本
```

处理步骤：

1. 把视频标题、简介、标签、字幕整理成一段视频内容文本。
2. 调用在线 AI 模型生成视频内容摘要。
3. 调用在线 AI 模型生成推荐关键词。
4. 调用 embedding 模型生成摘要 embedding。
5. 把 `title`、`summary`、`keywords` 写入业务库。
6. 把 `videoId`、`summaryEmbedding` 写入 Milvus。

最简单提示词：

```text
根据下面的视频内容，生成：
1. 视频内容摘要
2. 5 到 10 个用户可能搜索的关键词

视频内容：
{videoContent}
```

## 在线服务

在线服务在用户搜索时执行。

输入：

```text
用户搜索词
```

处理步骤：

1. 调用 embedding 模型，把用户搜索词转成 query embedding。
2. 用 query embedding 查询 Milvus，得到向量检索结果。
3. 用用户搜索词做关键词检索，匹配业务库里的 `title`、`keywords`、`summary`。
4. 合并向量检索结果和关键词检索结果。
5. 按最终分数倒序返回视频列表。

关键词检索最简单打分：

```text
title 命中：+5
keywords 命中：+4
summary 命中：+1
```

合并分数：

```text
finalScore = vectorScore * 0.7 + keywordScore * 0.3
```

如果 `keywordScore` 不是 0 到 1，需要先除以本次结果里的最高关键词分数。

## 执行步骤

1. 定义视频搜索数据结构，至少包含 `videoId`、`title`、`summary`、`keywords`。
2. 实现离线脚本或离线接口，输入视频内容，输出 `summary` 和 `keywords`。
3. 调用 embedding 模型生成 `summaryEmbedding`。
4. 把视频信息写入业务库，把摘要向量写入 Milvus。
5. 实现在线搜索接口，接收用户搜索词。
6. 在线搜索接口调用 embedding 模型生成 `queryEmbedding`。
7. 查询 Milvus，拿到向量召回结果。
8. 查询业务库，按关键词规则拿到关键词召回结果。
9. 合并两路结果，按 `finalScore` 排序。
10. 返回视频列表。

## 验收标准

离线验收：

- 给定一个视频内容，可以生成 `summary`。
- 给定一个视频内容，可以生成 5 到 10 个 `keywords`。
- 可以生成 `summaryEmbedding`。
- Milvus 中可以通过 `videoId` 查到对应向量记录。
- 业务库中可以通过 `videoId` 查到 `title`、`summary`、`keywords`。

在线验收：

- 用户搜索词可以生成 `queryEmbedding`。
- Milvus 可以返回语义相关的视频结果。
- 关键词检索可以命中 `title`、`keywords`、`summary`。
- 同一个 `videoId` 同时出现在两路结果时，只保留一条，并合并分数。
- 最终结果按 `finalScore` 从高到低排序。

效果验收：

- 搜索视频标题里的词，可以找到对应视频。
- 搜索关键词里的词，可以找到对应视频。
- 搜索和摘要语义相近但字面不完全一致的词，可以通过 Milvus 找到对应视频。
- 搜索无关内容时，结果为空或低相关结果排在后面。

## 测试

离线测试：

- 输入固定视频内容，验证能生成非空 `summary`。
- 输入固定视频内容，验证能生成非空 `keywords`。
- 输入固定 `summary`，验证能生成 embedding。
- 写入 Milvus 后，用相同或相近文本查询，验证能召回该 `videoId`。

在线测试：

- 构造 3 条测试视频数据。
- 搜索命中标题的词，验证目标视频排第一。
- 搜索命中关键词的词，验证目标视频排第一。
- 搜索语义相近的句子，验证向量检索能召回目标视频。
- 构造同一个视频同时被向量和关键词召回，验证最终只返回一次。
- 验证最终排序使用 `finalScore = vectorScore * 0.7 + keywordScore * 0.3`。

最小联调测试：

```text
准备 3 个视频
-> 跑离线服务写入业务库和 Milvus
-> 输入 3 个搜索词
-> 检查返回结果是否符合预期
```

## 最小版本完成定义

只要满足下面条件，就算第一版完成：

- 3 个测试视频可以离线入库。
- 每个视频都有 `summary`、`keywords`、`summaryEmbedding`。
- 用户输入搜索词后，可以同时得到向量结果和关键词结果。
- 两路结果可以合并排序。
- 搜索结果能返回正确的 `videoId` 列表。
