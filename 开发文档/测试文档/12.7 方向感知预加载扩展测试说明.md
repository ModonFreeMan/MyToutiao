# 12.7 方向感知预加载扩展测试说明

本文档记录 12.7 方向感知预加载扩展的测试范围和验收重点。

12.7 只验证“预加载谁”的候选选择升级，不扩展 PlayerController 的 preload 生命周期，不引入多槽位、不引入 preload queue，也不测试预渲染。

## 测试重点

```text
FeedPlaybackCoordinator 根据最近滑动方向选择单个 preload candidate。
unknown / 首屏保持 current + 1 语义。
forward 优先后方视频，后方无视频时 fallback 前方视频。
backward 优先前方视频，前方无视频时 fallback 后方视频。
非视频 item 不占用候选。
index 越界不抛错。
无候选时只清理 preload，不影响 active。
Feed items 非 append 替换会重置方向。
分页 append 不重置方向。
PlayerController 仍保持单 preload controller。
```

## FeedPlaybackCoordinator 单元测试

测试文件：

```text
test/unit/feed/feed_playback_coordinator_test.dart
```

新增或调整的关键用例：

```text
preloads current plus one video candidate:
  首屏 / unknown 方向保持 current + 1 语义。

forward direction falls back to previous video when no later video exists:
  已经形成 forward 方向后，后方没有视频时 fallback 到前方视频。

unknown direction does not fallback to previous video when no later video exists:
  首次 current change 落到尾部视频时，unknown 只查找后方视频，后方无视频则清理 preload。

backward direction selects previous video candidate:
  已经形成 backward 方向后，优先选择当前项之前的第一个视频。

backward direction falls back to next video when no previous video exists:
  backward 方向下前方没有视频时，fallback 到后方视频。

skips non-video items when selecting preload candidate:
  当前项或候选路径中出现图片时，继续寻找方向策略下的第一个视频。

out of range index does not throw:
  current index 越界时，候选选择安全返回，不抛异常。

feed items reset clears last scroll direction:
  Feed items 被非追加式替换后，旧列表高 index 到新列表低 index 不会误判为 backward。

non-append feed replacement clears last scroll direction:
  新列表不是旧列表 item id 前缀扩展时，方向状态会重置为 unknown。

feed pagination append does not clear last scroll direction:
  loadMore 只在尾部追加 items 时，不清理已有方向状态。

no direction-aware candidate clears preload without clearing active video state:
  无候选时只调用 disposePreload，active videoId 和 wantsToPlay 不被清空或修改。

direction-aware preload promotes when user scrolls into preloaded candidate:
  方向感知选中的 preload candidate 被滚动进入当前项时，仍复用 12.6 的 promote 逻辑。
```

重点断言：

```text
controller.preloadVideoId 指向方向策略下的预期候选。
无候选时 controller.preloadVideoId 为 null。
清理 preload 不会清空 active PlayerState.videoId。
清理 preload 不会修改 wantsToPlay。
非 append 替换后下一次 current change 按 unknown 语义选择候选。
分页 append 后下一次 current change 保留已有方向语义。
```

## PlayerController 回归范围

12.7 不新增 PlayerController 能力，只回归既有 preload 生命周期和 promote 行为：

```text
preloadVideo 重复候选由 PlayerController 内部跳过或按既有 failed 重试语义处理。
disposePreload 只清理 preload controller / preload slot。
preload success 不修改 PlayerState。
preload success 不修改 wantsToPlay。
preload success 不调用 play。
playVideo 命中 preloaded controller 时继续复用 12.6 promote 逻辑。
```

不新增以下测试：

```text
多 preload controller。
preload queue 并发初始化。
current + 2 和 current - 1 同时预加载。
预渲染或首帧缓存。
多候选 dispose 策略。
```

重点观察：

```text
Feed 首屏自动播放正常。
连续向下滑动时后方视频可被 preload 并 promote。
向上回滑时前方视频优先成为 preload candidate。
视频 / 图文混排时 candidate 正确跳过非视频。
Feed 数据刷新或搜索结果切换后，首个 current change 按 unknown 处理。
用户主动暂停后 preload success 不恢复播放。
清晰度切换、横屏播放、Feed 覆盖返回无回归。
```
