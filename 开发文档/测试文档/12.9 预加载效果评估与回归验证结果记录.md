说明：

```text
本次样本少于 12.9 建议的 30 个 video visible 样本。
preload only 的 valid_initialize_samples 只有 3 个，initialize_ms 对比只能作为参考。
本次结果可以作为方向性结论，不能作为最终硬性性能验收。
```

核心指标对比：

```text
startup_ms:
  p50: baseline 53ms -> preload only 0ms，下降 53ms
  p90: baseline 686ms -> preload only 0ms，下降 686ms
  p95: baseline 686ms -> preload only 0ms，下降 686ms

first_frame_ms:
  p50: baseline 252ms -> preload only 204ms，下降 48ms
  p90: baseline 890ms -> preload only 213ms，下降 677ms
  p95: baseline 890ms -> preload only 213ms，下降 677ms

  p50: baseline 104ms -> preload only 1717ms，上升 1613ms
  p90: baseline 1681ms -> preload only 2203ms，上升 522ms
  p95: baseline 2550ms -> preload only 2203ms，下降 347ms

startup_buffering_count:
  p50: baseline 0 -> preload only 0，持平
  p90: baseline 1 -> preload only 1，持平
  p95: baseline 1 -> preload only 2，上升 1

startup_buffering_total_ms:
  p50: baseline 0ms -> preload only 0ms，持平
  p90: baseline 113ms -> preload only 119ms，上升 6ms
  p95: baseline 366ms -> preload only 279ms，下降 87ms
```

预加载命中解释：

```text
preload_visible_items: 12
preload_hits: 11
preload_misses: 1
preload_promoted_to_active: 11
preload_hit_rate: 91.7%
```

命中解释判断：

```text
preload_promoted_to_active 与 preload_hits 完全一致。
preload_hit_rate 为 91.7%，能够解释 startup_ms p90 / p95 从 686ms 降到 0ms 的收益。
ignored_late_events 从 baseline 8 降到 preload only 6，未观察到迟到事件异常放大。
initialize_failed_sessions 均为 0，没有初始化失败回归。
```

阶段结论：

```text
本次采样显示，preload only 相比 baseline 在 startup_ms p90 / p95 上有非常明显下降。
first_frame_ms p90 / p95 同样明显下降，说明用户可感知的首帧呈现长尾也改善。
preload hit rate 高，且 promoted 数与 hit 数一致，收益来源可以被 preload 命中热切换解释。
buffering p90 基本持平，p95 总时长下降，但 p95 次数从 1 上升到 2，需要后续更大样本确认。
initialize_ms 因 preload only 有效样本只有 3 个，暂不能判定是否恶化。
```

后续建议：

```text
继续采集至少 30 个 video visible 样本，优先补足 preload only 的 valid_initialize_samples。
如果扩大样本后 startup_ms p90 / p95 仍稳定下降，且 buffering 与 initialize_ms 未明显恶化，可以认为 12.9 性能收益验收通过。
如果 initialize_ms p90 / p95 持续上升，需要复查 active 初始化是否被后台 preload 竞争资源影响。
```
