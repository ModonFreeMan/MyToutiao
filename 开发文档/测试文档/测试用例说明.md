## 22. 起播性能指标测试

测试文件：

[playback_startup_metrics_test.dart](D:/Flutter/VideoPlayer/test/unit/player/playback_startup_metrics_test.dart:1)

测试内容：

- 完整 startup session 可以输出 `first_frame_ms`、`startup_ms`、`initialize_ms`
- `markXXX` 公共 API 只接收已绑定的 `PlaybackStartupSessionRef`
- 不属于当前 active session 的首帧、播放确认和 buffering 事件会被忽略
- expired session 的 initialize success 仍可计入 `initialize_ms`
- 初始化失败 session 计入 `initialize_failed_sessions`，且不进入初始化成功样本
- 重复事件不会覆盖首次时间点，并计入 `ignored_late_events`
- 重复播放请求不会覆盖首次 `play_requested_at`，但会累计 `play_request_sources`
- 同一 session 内重复 initialize 事件不会覆盖首次 `initialize_ms` 样本
- 未闭合 buffering span 会按 report 时间和起播后窗口进行截断
- 未启用预加载时输出 `preload_enabled = false`、`preload_hit_rate = null`、`preload_hit_rate_label = N/A`
- 空样本时 p50 / p90 / p95 输出 `null`
- 无效 session ref、重复关闭和迟到事件不会向调用方抛出异常

通过标准：

```text
PlaybackStartupMetrics 只做内存采集，不影响播放链路；
所有指标样本都以 startup session 归属，迟到或重复事件不会串样本；
baseline report 可以稳定输出样本数、丢弃计数、分位数和预加载 N/A 元信息。
播放请求来源计数可以辅助后续区分自动起播、恢复播放、横屏恢复等触发路径。
非 Feed startup session 链路不会产生 baseline 样本。
```
