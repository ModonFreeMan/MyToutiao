# 12.9 预加载效果评估与回归验证测试说明

本文档记录 12.9 的测试范围、手工评估流程和回归验证命令。

12.9 不新增 preload 策略，只验证现有单 preload + 方向感知 + 快速滑动节流是否带来可解释收益，并确认既有播放链路无回归。

## 测试重点

```text
report JSON 字段完整。
baseline 无 preload 样本时显示 N/A。
preload only 可以输出 hit / miss / promoted / hit rate。
startup_ms / initialize_ms / first_frame_ms / buffering 分位数口径不变。
debug-only report 导出入口不参与播放决策。
快速滑动下 pending preload debounce 仍然有效。
旧 preload 迟到不会覆盖当前 preload。
用户暂停、横屏、清晰度切换和 Feed 覆盖返回无回归。
```

## 手工评估流程

同一设备、同一 Feed 数据源、同一网络条件下分别采集：

```text
baseline:
  关闭 preload 或使用未开启 preload 的版本。
  至少 30 个 video visible 样本。

preload only:
  开启 12.6 / 12.7 / 12.8。
  至少 30 个 video visible 样本。
```

推荐路径：

```text
1. 冷启动进入 Feed。
2. 等首屏视频进入播放。
3. 匀速向下滑动，覆盖视频和图文混排。
4. 向上回滑，覆盖方向感知候选。
5. 快速连续滑动，覆盖 debounce 和过期 preload。
6. 停在视频上主动暂停，再继续滑动。
7. 进入搜索覆盖页后返回 Feed。
8. 执行一次清晰度切换。
9. 进入横屏播放并返回竖屏。
10. 通过 debug report 导出 JSON。
```

记录信息：

```text
代码版本或 commit。
运行模式：baseline / preload only。
设备或模拟器信息。
Feed 数据来源。
样本数量。
是否发生初始化失败或明显网络波动。
report JSON 摘要。
```

## 数据通过口径

先确认样本质量：

```text
valid_startup_samples 足够。
valid_initialize_samples 足够。
incomplete_sessions 不异常偏高。
ignored_late_events 不异常偏高。
initialize_failed_sessions 不异常偏高。
```

再判断收益：

```text
preload only 的 startup_ms p90 或 p95 有可解释下降。
preload_hit_rate 大于 0。
preload_promoted_to_active 与 preload_hits 一致或接近。
initialize_ms p90 / p95 没有明显恶化。
startup_buffering_count / startup_buffering_total_ms 没有明显恶化。
没有新增崩溃、未捕获异常或 controller 泄漏迹象。
```

本阶段不设置硬编码百分比阈值，优先保证同口径、可重复、可解释。
