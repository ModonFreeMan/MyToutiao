# 12.8 快速滑动预加载节流与释放收敛测试说明

本文档记录 12.8 快速滑动预加载节流与释放收敛的测试范围、关键用例和回归验证命令。

12.8 只验证 FeedPlaybackCoordinator 内部的单 preload 调度节流，不扩展 PlayerController preload 生命周期，不引入多槽位、不引入 preload queue，也不测试 dispose 队列。

本阶段的测试目标是：

```text
快速滑动时，未过稳定窗口的旧 preload candidate 不会启动 preload。
current change 入口会先取消旧 pending preload。
active 播放 / stop 流程不等待 preload debounce。
无候选和换源时会及时清理 preload 业务槽位。
换源 + 无候选不会重复请求 disposePreload。
provider dispose 后 pending timer 不会继续启动 preload。
PlayerController 的 preload 过期隔离和 promote 逻辑保持不变。
```

## FeedPlaybackCoordinator 单元测试

测试文件：

```text
test/unit/feed/feed_playback_coordinator_test.dart
```

新增或调整的关键用例：

```text
rapid current changes only schedule latest preload after delay:
  连续 current change 时，旧 pending preload 在 120ms 稳定窗口内被取消。
  只有最后一次 current change 的候选会触发 preloadVideo。

candidate null cancels pending preload and disposes preload immediately:
  新列表没有 preload candidate 时，入口处取消 pending preload 并清理 preload。
  等待稳定窗口后，不会启动旧 candidate。

feed replacement cleanup does not block latest preload debounce registration:
  Feed items 非 append 替换时，旧 preload 业务槽位被清理。
  清理使用 fire-and-forget，不阻塞新 candidate 注册 debounce。
  稳定窗口结束后，新列表 candidate 正常进入 preload。

container dispose cancels pending preload timer:
  provider / container dispose 后，pending timer 被取消。
  等待稳定窗口后，不会创建新的 preload controller。

slow preload initialization does not block current feed change:
  preload 初始化 hold 住时，active 播放链路仍可完成。
  preload 进入 initializing，但不阻塞 current change 的 active 起播。

rapid current changes keep only latest preload candidate visible:
  旧 preload initializing 迟到后，不会覆盖最新 preload 状态。
  最终有效 preload 仍指向最新候选。
```

同步调整的既有用例：

```text
preloads current plus one video candidate:
  等待 preload debounce 窗口后再断言 preload 状态。

current change promotes preloaded next video and schedules following preload:
  promote 后继续等待 debounce，再断言后续 preload candidate。

forward / backward direction 相关用例:
  方向候选语义不变，但断言前需要等待 debounce 窗口。
```

重点断言：

```text
稳定窗口结束前不会创建 preload controller。
快速连续滑动后只有最新候选触发 preloadVideo。
无候选时 preloadVideoId 为 null。
换源后旧列表 candidate 不会迟到启动 preload。
换源清理旧 preload 不阻塞新 candidate debounce 注册。
provider dispose 后 timer 不会再触发 preloadVideo。
当前 active videoId 不因 disposePreload 被清空。
```

## PlayerController 回归范围

12.8 不新增 PlayerController 能力，保留 12.5 / 12.6 回归即可：

```text
old preload success late does not overwrite latest preload。
old preload failure late does not overwrite latest preload。
disposePreload invalidates pending initialization and disposes once。
preload success / failure does not modify PlayerState。
preload success / failure does not modify wantsToPlay。
playVideo promotes matching preloaded controller to active。
preload initializing is a miss and falls back to active initialization。
forceRestart bypasses preload without counting miss。
```

不新增以下测试：

```text
多 preload controller。
preload queue。
dispose 队列。
平台层强取消 initialize。
FeedPlaybackCoordinator 内部 timer 引用白盒断言。
FeedPlaybackCoordinator 内构造 preload 初始化失败归属。
```

其中 preload 初始化失败归属继续由 PlayerController 单元测试覆盖。
