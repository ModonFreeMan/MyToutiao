# 今日头条视频播放流 + 搜索模块 MVP 框架设计说明文档

## 1. 技术栈

| 类别 | 选型 |
| --- | --- |
| 开发框架 | Flutter |
| 状态管理 | Riverpod |
| 播放能力 | Flutter 视频播放器插件 / 系统播放器封装 |
| 本地存储 | SharedPreferences |
| 数据来源 | 本地 Mock 数据 |


## 2. 功能范围

### 2.1 视频流功能

- App 启动后直接进入内容流页面
- 全屏沉浸式内容流
- 上下滑切换内容
- 视频自动播放
- 视频暂停 / 继续播放
- 播放进度展示
- 拖动进度条调整播放进度
- 数据分页加载
- 视频预加载
- 视频预渲染
- 清晰度切换
- 横屏播放
- 竖屏恢复
- 视频流中支持图片卡

### 2.2 互动展示功能

- 展示头像
- 展示作者名称
- 展示标题
- 展示点赞按钮
- 展示评论按钮
- 展示收藏按钮
- 展示分享按钮
- 互动按钮只展示 UI，不处理真实点击逻辑

### 2.3 搜索功能

- 视频流顶部展示搜索框
- 点击搜索框进入搜索中间页
- 搜索中间页支持输入关键词
- 支持通过键盘搜索触发搜索
- 支持保存搜索历史
- 支持展示搜索历史
- 支持点击搜索历史再次搜索
- 搜索结果页展示视频类搜索结果
- 点击搜索结果跳转到内容流并播放对应视频

### 2.4 推荐词功能

- 视频流底部展示相关搜索推荐词
- 推荐词根据当前内容匹配
- 点击推荐词进入搜索结果页
- 使用推荐词作为搜索关键词发起搜索


## 3. 总体架构设计

项目采用 Flutter + Riverpod 的分层架构。

```text
Page / Widget 页面层
        ↓
Coordinator / Orchestrator 业务编排层
        ↓
ViewModel / Notifier 状态管理层
        ↓
UseCase / Service 业务服务层
        ↓
Repository 数据仓库层
        ↓
DataSource 数据源层
```

### 3.1 分层职责

#### 页面层

负责 UI 展示、用户交互、页面跳转。

#### 业务编排层

负责跨模块业务流程编排，例如内容流当前项变化后的播放 / 停止决策、搜索结果点击后的 Feed 定位与播放器触发、Feed 被搜索页覆盖前后的暂停 / 恢复。

#### 状态管理层

负责页面状态维护、事件处理、异步数据加载。

#### 业务服务层

负责播放器控制、推荐词匹配、搜索匹配、性能统计等业务逻辑。

#### 数据仓库层

负责统一封装数据访问，对上层屏蔽数据来源。

#### 数据源层

负责提供本地 Mock 数据、本地缓存数据。

### 5.8 recommendation 模块

#### 职责

- 根据当前内容匹配推荐词
- 向 FeedPage 提供底部相关搜索词
- 支持推荐词点击后发起搜索

#### 核心服务

```text
RecommendationService
```

#### 推荐词匹配来源

- 当前内容标题
- 当前内容描述
- 当前内容标签
- 当前内容绑定的推荐词

#### 推荐词匹配优先级

1. 优先使用 FeedItem 直接绑定的 recommendationWords
2. 如果没有绑定，则根据 tags 匹配推荐词库
3. 如果 tags 没有命中，则根据 title 关键词匹配
4. 如果 title 没有命中，则返回默认推荐词

### 5.9 performance 模块

#### 职责

- 记录播放器初始化耗时
- 记录首帧渲染耗时
- 记录切换视频起播耗时
- 记录预加载命中情况
- 记录预渲染命中情况
- 统计优化前后起播耗时
- 计算优化幅度
- 输出性能统计结果

#### 核心服务

```text
PerformanceService
```

#### 核心指标

```text
coldStartFirstFrameTime
switchVideoFirstFrameTime
preloadHitFirstFrameTime
preloadMissFirstFrameTime
preRenderHitFirstFrameTime
averageFirstFrameTimeBeforeOptimization
averageFirstFrameTimeAfterOptimization
optimizationRate
```

#### 优化幅度计算方式

```text
optimizationRate =
  (beforeOptimizationTime - afterOptimizationTime)
  / beforeOptimizationTime
  * 100%
```

#### 示例统计结果

```text
未启用预加载 / 预渲染：
平均切换起播耗时：850ms

启用预加载 / 预渲染后：
平均切换起播耗时：320ms

优化幅度：
(850 - 320) / 850 * 100% = 62.35%
```

## 10. 核心业务流程

```text

  ↓

  ↓

  ↓

  ↓
开始播放
  ↓
预加载下一个视频
```

### 10.2 内容流分页流程

```text
进入 FeedPage
  ↓
请求第一页 Feed 数据
  ↓
渲染 PageView
  ↓
用户上下滑动
  ↓
更新 currentIndex
  ↓
判断是否接近列表底部
  ↓
如果接近底部，加载下一页
  ↓
追加到 FeedState.items
```

### 10.3 视频切换播放流程

```text
用户滑动到新内容
  ↓
FeedPage 通知 FeedPlaybackCoordinator 当前项变化
  ↓
判断内容类型
  ↓
如果是图片卡，停止播放器
  ↓
如果是视频卡，触发 PlayerController 播放当前视频
  ↓
检查当前视频是否已预加载
  ↓
命中预加载则直接播放
  ↓
未命中预加载则初始化播放器
  ↓
记录首帧耗时
  ↓
播放当前视频
  ↓
预加载下一个视频
  ↓
释放远离当前索引的视频播放器
```

### 10.4 视频预加载 / 预渲染流程

```text
当前视频开始播放
  ↓
获取 currentIndex + 1 的内容
  ↓
判断是否为视频内容
  ↓
如果是视频，创建预加载播放器实例
  ↓
加载视频资源
  ↓
准备首帧
  ↓
标记为 preloadReady / preRenderReady
  ↓
用户滑动到该视频
  ↓
直接复用已准备好的播放器
  ↓
降低起播耗时
```

### 10.5 清晰度切换流程

用户点击清晰度按钮

用户选择目标清晰度

## 13. 性能优化设计

### 13.1 起播速度优化目标

- 降低首个视频首帧耗时
- 降低上下滑切换视频后的起播耗时
- 降低视频初始化等待时间
- 提升预加载命中率
- 通过数据展示优化幅度

### 13.2 优化手段

- 当前视频优先初始化
- 下一个视频提前预加载
- 下一个视频提前预渲染首帧
- 复用已预加载的播放器实例
- 释放远离当前索引的视频播放器
- 图片卡不创建播放器
- 分页数据提前加载
- 视频封面提前缓存

### 13.3 播放器缓存策略

```text
当前索引：currentIndex
预加载索引：currentIndex + 1
保留索引：currentIndex - 1、currentIndex、currentIndex + 1
释放索引：距离 currentIndex 超过 1 的视频播放器
```

### 13.4 性能统计指标

```text
videoInitStartTime
videoReadyTime
firstFrameRenderTime
switchStartTime
playStartTime
preloadStartTime
preloadReadyTime
preRenderReadyTime
preloadHit
preRenderHit
```

### 13.5 优化数据展示

技术文档中展示以下数据：

- 未优化平均起播耗时
- 优化后平均起播耗时
- 预加载命中率
- 预渲染命中率
- 平均节省耗时
- 优化幅度百分比

示例：

```text
未优化平均切换起播耗时：850ms
优化后平均切换起播耗时：320ms
平均节省耗时：530ms
优化幅度：62.35%
```

## 17. 架构总结

本项目使用 Flutter + Riverpod 实现，整体采用分层架构。

核心模块包括：

- `feed`：负责内容流展示、分页、视频卡、图片卡
- `player`：负责播放、暂停、进度、清晰度、横屏、预加载、预渲染
- `search`：负责搜索中间页、搜索历史、搜索结果
- `feed/coordinators`：负责跨模块业务流程编排，例如 Feed 播放切换、搜索结果定位播放、页面覆盖时的暂停 / 恢复
- `recommendation`：负责相关搜索推荐词匹配
- `performance`：负责起播性能统计和优化幅度计算
- `storage`：负责本地搜索历史和轻量配置存储
- `data`：负责数据模型、仓库和数据源
- `core`：负责全局基础能力
- `common`：负责通用组件

整体数据流保持单向依赖：

```text
UI Page
  ↓
Coordinator / Orchestrator
  ↓
Riverpod ViewModel / Notifier
  ↓
Service / Repository
  ↓
DataSource
```

该架构可以覆盖 MVP 的全部必做功能，同时支持后续扩展真实接口、AI 推荐词生成、语义搜索、更多内容卡片类型和更完整的播放器能力。
